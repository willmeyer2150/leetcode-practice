## Initial Thoughts

My first thought was to solve this using a nested loop structure. This is the brute force method and it does work. I know that there is a better way to solve using an unordered map. I am not very developed with that kind of data structure, so I chose to solve it the brute force method first, and then to optimize my solution after I understand it.

## Key Insight

If solving the brute force method, the key thing to understand is that index i tracks each individual value in the array, and j tracks the values which create all combinations needing testing.

If you take a small example (3-4 values), you can easily write out all combinations in order and you will see how the indexes need to move to provide a solution.
